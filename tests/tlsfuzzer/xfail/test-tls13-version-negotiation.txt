#
# tlsfuzzer's `test-tls13-version-negotiation.py` enumerates every
# pre-RFC TLS 1.3 draft codepoint (draft0..draft28) and every
# bogus-version codepoint (TLS 1.4-1.8, SSL 3.0 etc.) to verify that
# the server reacts the way OpenSSL historically did. We don't:
#
# - We never advertised draft codepoints (RFC 8446 final is the only
#   `supported_versions` value we negotiate). When tlsfuzzer offers
#   draft N alone, we either reject or fall through to TLS 1.2 in
#   ways tlsfuzzer doesn't expect.
# - We don't try to interpret bogus future-version codepoints in the
#   `legacy_version` field; we trust `supported_versions` exclusively.
#
# Both are spec-conforming choices, just not the OpenSSL-flavoured
# ones tlsfuzzer codifies. XFAIL the lot — none represents a real
# bug or security gap in our implementation.
