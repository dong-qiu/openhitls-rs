# Phase T127 — `test-tls13-dhe-shared-secret-padding.py` XFAILs (3 entries).
#
# This script exercises (EC)DHE shared secrets across many curves and
# leading-zero-padding edge cases (the class of bug where a stripped
# leading zero in the shared secret diverges the key schedule). It
# passes overwhelmingly — 513 PASS — including all secp256r1 / secp384r1
# / secp521r1 / x25519 conversations (P-384/P-521 ECDHE was wired up in
# Phase I99).
#
# 3 conversations fail, each because the named group is not in the
# server's advertised `supported_groups` — a structural gap, not a
# padding bug:
#
# - `TLS 1.3 with ffdhe2048` / `TLS 1.3 with ffdhe3072`: RFC 7919
#   finite-field DHE groups. The TLS layer does not negotiate FFDHE
#   named groups yet — that is the planned FFDHE phase (the
#   `hitls-crypto` DH primitive exists; the supported_groups / key
#   exchange wiring does not).
# - `TLS 1.3 with x448`: X448 ECDHE works (the `KeyExchange` supports
#   it), but X448 is not in the `s-server` default `supported_groups`,
#   so a client offering only x448 gets handshake_failure. Adding x448
#   to the default group list would close this.
